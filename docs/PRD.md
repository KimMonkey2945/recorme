# record MVP PRD

> ⚠️ **이 문서는 고정 명세가 아니라 현재 시점의 기준 스냅샷이다.** 범위·기능·데이터 모델·기술 스택은 개발 진행과 요구사항 변화에 따라 **얼마든지 변경될 수 있다.** 변경 시 `docs/`(api-contract·backend·database 등)와 함께 정합을 맞춘다.

## 핵심 정보

**목적**: 날짜별로 하루를 글로 기록하고 언제든 다시 꺼내볼 수 있는 개인 모바일 일기장
**사용자**: 매일 짧은 글쓰기로 하루를 정리하고 싶은 모바일 사용자

## 사용자 여정

```
1. 앱 실행
   ↓ 로컬 토큰 유무 확인

   [토큰 없음 / 만료] → 로그인 페이지
   [토큰 유효]        → 메인 페이지 (캘린더)

2. 로그인 페이지
   ↓ 소셜 로그인 버튼 탭 (카카오 / 구글)

   [최초 진입 - 미가입] → Supabase 세션 발급 → (백엔드 첫 요청 시 자동 가입) → 메인 페이지
   [재진입 - 기가입]    → Supabase 세션 발급 → 메인 페이지
   [인증 실패]          → 에러 토스트 후 로그인 페이지 유지

3. 메인 페이지 (캘린더)
   ↓ 날짜 탭 또는 하단 탭 이동

   [날짜 탭 - 해당 날짜 일기 없음] → 글 에디터 페이지 (신규 작성 모드)
   [날짜 탭 - 해당 날짜 일기 있음] → 글 상세 페이지
   [하단 목록 탭]                   → 글 목록 페이지

4. 글 에디터 페이지
   ↓ 사용자 행동

   [등록 버튼]          → DRAFT 저장(미확정·수정가능·미분석) → 메인 페이지
   [오늘을 기억하기 버튼] → 확인 다이얼로그 → 확정(PENDING·감정분석 1회·이후 수정 불가) → 상세 페이지
   [취소 버튼]          → 이전 페이지 (메인 페이지 또는 글 상세 페이지)

5. 글 상세 페이지
   ↓ 사용자 행동

   [수정 버튼] → 글 에디터 페이지 (수정 모드, 기존 내용 로드)
   [삭제 버튼] → 삭제 확인 다이얼로그 → 확인 시 소프트 삭제 → 메인 페이지
   [뒤로가기]  → 이전 페이지 (글 목록 페이지 또는 메인 페이지)

6. 글 목록 페이지
   ↓ 사용자 행동

   [항목 탭]     → 글 상세 페이지
   [캘린더 탭]   → 메인 페이지
   [로그아웃]    → 토큰 삭제 → 로그인 페이지
```

## 기능 명세

### 1. MVP 핵심 기능

| ID | 기능명 | 설명 | MVP 필수 이유 | 관련 페이지 |
|----|--------|------|--------------|------------|
| **F001** | 로그인 (소셜/이메일) | Supabase Auth로 소셜 로그인(카카오/구글) 또는 이메일 가입·로그인(확인 메일 필수), Supabase 세션 발급. 백엔드는 Supabase JWT 검증 + 최초 요청 시 자동 가입(이메일·소셜 동일 경로) (애플은 추후 확장) | 서비스 진입의 유일한 인증 관문 | 로그인 페이지 |
| **F002** | 캘린더 날짜 선택 | 월별 캘린더 표시, 일기 작성 여부를 날짜별로 시각 표시, 날짜 탭으로 작성·조회 진입 | 날짜 기반 일기 작성의 핵심 진입점 | 메인 페이지 |
| **F003** | 일기 작성 | 선택된 날짜에 자유 텍스트 일기 작성(본문 **최대 500자 하드 제한** — LLM 감정분석 비용·품질 가드). 해당 날짜 일기 미존재 시 신규 생성(INSERT). **[등록]**은 DRAFT(미확정·수정가능)로 저장, **[오늘을 기억하기]**는 확정(이후 수정 불가, 감정분석은 확정 시 1회 — 분석 자체는 Phase 4) | 서비스의 핵심 행위 | 글 에디터 페이지 |
| **F004** | 일기 목록 조회 | 작성된 일기를 날짜 역순으로 나열. 날짜와 내용 미리보기 표시, 커서 기반 무한 스크롤 | 과거 기록 탐색의 기본 수단 | 글 목록 페이지 |
| **F005** | 일기 단건 조회 | 특정 날짜 일기의 전체 내용 표시 | 작성한 글을 읽는 기본 기능 | 글 상세 페이지 |
| **F006** | 일기 수정 | 기존 일기 내용 수정(UPDATE). **DRAFT(미확정) 일기만 수정 가능**, 같은 날짜에 글 에디터 재진입 시에도 DRAFT면 수정으로 처리. **확정('오늘을 기억하기') 후에는 수정 불가**(삭제는 가능 → 같은 날짜 재작성) | 하루 1기록 원칙 구현, 오탈자·내용 변경 지원 | 글 에디터 페이지 |
| **F007** | 일기 삭제 | 일기 소프트 삭제(deleted_at 타임스탬프 기록). 확인 다이얼로그 후 처리. 삭제된 날짜는 다시 작성 가능. 첨부 사진 행·디스크 파일은 즉시 회수(복원 불가) | 잘못 작성된 기록 제거·디스크 공간 절약 | 글 상세 페이지 |
| **F012** | 일기 사진 첨부 | 일기에 사진 첨부(일기당 **최대 5장·장당 5MB**, jpg/png/webp). 바이너리는 스토리지에 저장하고 DB엔 경로만 보관(`POST /diaries/{id}/images`·`DELETE …/images/{imageId}`). 개별/일기 삭제 시 디스크 파일 즉시 회수 | 글과 함께 그날의 사진 기록 | 글 에디터/상세 페이지 |
| **F011** | 프로필 조회·수정 | 내 프로필(닉네임·프로필 이미지·자기소개 bio) 조회 및 수정. `GET/PUT /users/me`. 가입 시 JIT로 생성된 정보를 사용자가 직접 갱신 | 가입 정보의 사용자 관리·표시 이름 변경 | 프로필 페이지 |

### 2. MVP 필수 지원 기능

| ID | 기능명 | 설명 | MVP 필수 이유 | 관련 페이지 |
|----|--------|------|--------------|------------|
| **F010** | 로그아웃 + 세션 관리 | 로그아웃 시 Supabase signOut(세션 삭제). access 만료 시 Supabase SDK가 자동 갱신, Dio 인터셉터는 Supabase access token 첨부만 담당 | 인증 세션 유지 및 안전한 종료 | 메인 페이지, 글 목록 페이지 |

### 3. MVP 이후 기능 (제외)

- 감정 분석(LLM) - 일기 내용 기반 감정 자동 분류
- 감정 기반 테마 - 배경·필체 자동 적용
- 감정 기반 음악 - 기분에 맞는 음악 재생
- 공유 기능 - 공개 범위 설정, 공유 링크 생성
- 피드 - 다른 사용자 글 탐색
- 친구 관계 - 팔로우/팔로잉
- 공감(리액션) - 다른 사용자 글에 반응
- 설정 - 테마, 언어, 알림 설정
- 푸시 알림

## 페이지별 상세 기능

### 로그인 페이지

> **구현 기능:** `F001` | **메뉴 위치:** 비로그인 시 앱 자동 리디렉션

| 항목 | 내용 |
|------|------|
| **역할** | 소셜 인증 전용 진입 게이트. 앱의 모든 기능 이용 전 반드시 통과하는 단일 인증 화면 |
| **진입 경로** | 앱 실행 시 토큰 없거나 만료된 경우 자동 이동. 로그아웃 후 자동 이동 |
| **사용자 행동** | 카카오, 구글 중 하나의 소셜 로그인 버튼을 탭하여 소셜 SDK 인증 흐름 진행 |
| **주요 기능** | • 카카오 로그인 버튼 (Supabase signInWithOAuth)<br>• 구글 로그인 버튼 (google_sign_in → Supabase signInWithIdToken)<br>• 이메일 로그인 폼 + "이메일로 회원가입"(닉네임·이메일·비밀번호 → Supabase signUp, 확인 메일 발송 후 안내 화면)<br>• Supabase Auth가 검증·세션(JWT) 발급, SDK가 토큰 저장·자동 갱신<br>• 백엔드 첫 요청 시 미가입 사용자 자동 가입(JIT, 닉네임·이메일 세팅)<br>• (애플 로그인은 추후 확장) |
| **다음 이동** | 로그인 성공 → 메인 페이지, 인증 실패 → 에러 토스트 표시 후 로그인 페이지 유지 |

---

### 메인 페이지

> **구현 기능:** `F002`, `F010` | **메뉴 위치:** 하단 내비게이션 바 - 캘린더 탭

| 항목 | 내용 |
|------|------|
| **역할** | 앱의 홈 화면. 월별 캘린더로 날짜별 일기 작성 여부를 한눈에 파악하고 작성·조회 진입점 제공 |
| **진입 경로** | 로그인 성공 후 자동 이동. 글 에디터 페이지 저장/취소 후 복귀. 하단 내비게이션 캘린더 탭 탭. 삭제 완료 후 복귀 |
| **사용자 행동** | 월별 캘린더를 좌우 스와이프로 월 이동, 날짜를 탭하여 글 작성 또는 조회로 진입, 로그아웃 실행 |
| **주요 기능** | • 월별 캘린더 표시 (이전/다음 달 스와이프 이동)<br>• 일기가 작성된 날짜에 점(dot) 시각 표시<br>• 오늘 날짜 하이라이트<br>• 날짜 탭 → 일기 없으면 글 에디터 페이지(신규 모드), 일기 있으면 글 상세 페이지<br>• 상단 앱 바 로그아웃 버튼 (F010, 토큰 삭제 후 로그인 페이지 이동) |
| **다음 이동** | 날짜 탭(일기 없음) → 글 에디터 페이지, 날짜 탭(일기 있음) → 글 상세 페이지, 로그아웃 → 로그인 페이지 |

---

### 글 에디터 페이지

> **구현 기능:** `F003`, `F006` | **메뉴 위치:** 메인 페이지 날짜 탭(일기 없음) 또는 글 상세 페이지 수정 버튼

| 항목 | 내용 |
|------|------|
| **역할** | 일기 신규 작성(F003)과 기존 수정(F006)을 단일 화면에서 처리하는 에디터. 모드에 따라 초기 내용 여부만 다름 |
| **진입 경로** | 메인 페이지에서 일기 없는 날짜 탭 시 신규 작성 모드로 진입. 글 상세 페이지 수정 버튼 탭 시 수정 모드로 진입 |
| **사용자 행동** | 날짜를 확인한 후 텍스트 필드에 내용을 입력하고 저장, 또는 취소로 이전 화면 복귀 |
| **주요 기능** | • 선택된 날짜 표시 (에디터 내 변경 불가)<br>• 멀티라인 자유 텍스트 입력 필드<br>• 수정 모드 진입 시 기존 일기(DRAFT) 내용 자동 로드<br>• **등록** 버튼: DRAFT로 저장(미확정·수정가능·미분석). 해당 날짜 일기 미존재 시 INSERT, 존재(DRAFT) 시 UPDATE (하루 1기록 원칙)<br>• **오늘을 기억하기** 버튼: 확인 다이얼로그 후 확정(PENDING). 감정분석 1회 트리거(분석 자체는 Phase 4), **확정 후 수정 불가**<br>• **취소** 버튼: 변경 사항 미저장 후 이전 화면 복귀 |
| **다음 이동** | 등록 성공 → 메인 페이지, 확정('오늘을 기억하기') → 상세 페이지(분석 중 표시), 취소 → 이전 화면 (메인 페이지 또는 글 상세 페이지) |

---

### 글 목록 페이지

> **구현 기능:** `F004`, `F010` | **메뉴 위치:** 하단 내비게이션 바 - 목록 탭

| 항목 | 내용 |
|------|------|
| **역할** | 사용자가 작성한 모든 일기를 날짜 역순으로 나열하여 탐색하는 아카이브 화면 |
| **진입 경로** | 하단 내비게이션 바 목록 탭 탭 |
| **사용자 행동** | 목록을 스크롤하여 과거 일기를 탐색하고, 항목을 탭하여 글 상세 페이지로 이동 |
| **주요 기능** | • 일기 목록을 날짜 역순으로 표시<br>• 각 항목: 날짜 헤더, 내용 미리보기 (최대 2줄 말줄임)<br>• 항목 탭 → 글 상세 페이지 이동<br>• 커서 기반 무한 스크롤 페이징 (OFFSET 미사용)<br>• 상단 앱 바 로그아웃 버튼 (F010) |
| **다음 이동** | 항목 탭 → 글 상세 페이지, 로그아웃 → 로그인 페이지 |

---

### 글 상세 페이지

> **구현 기능:** `F005`, `F007` | **메뉴 위치:** 글 목록 페이지 항목 탭 또는 메인 페이지 날짜 탭(일기 있음)

| 항목 | 내용 |
|------|------|
| **역할** | 특정 날짜 일기의 전체 내용을 표시하고 수정/삭제 액션을 제공하는 뷰어 화면 |
| **진입 경로** | 글 목록 페이지에서 항목 탭. 메인 페이지에서 일기가 존재하는 날짜 탭 |
| **사용자 행동** | 일기 전체 내용을 읽고, 필요 시 수정 버튼으로 에디터 이동 또는 삭제 버튼으로 일기 제거 |
| **주요 기능** | • 날짜 헤더와 일기 전체 내용 스크롤 표시<br>• **수정** 버튼: 글 에디터 페이지 수정 모드로 이동 (기존 내용 전달)<br>• **삭제** 버튼: 삭제 확인 다이얼로그 표시 후 확인 시 소프트 삭제(deleted_at 기록, 같은 날짜 재작성 허용)<br>• 뒤로가기로 이전 화면 복귀 |
| **다음 이동** | 수정 버튼 → 글 에디터 페이지, 삭제 완료 → 메인 페이지, 뒤로가기 → 이전 화면 |

## 데이터 모델

### users (사용자)

| 필드 | 설명 | 타입/관계 |
|------|------|----------|
| id | 내부 PK | BIGINT |
| uuid | 외부 노출용 식별자(공유 등) | UUID |
| supabase_uid | Supabase auth.users.id 매핑(JWT sub) | UUID |
| nickname | 표시 이름 | VARCHAR |
| email | 소셜/이메일 제공 이메일 | VARCHAR |
| profile_image_url | 프로필 이미지 URL | VARCHAR |
| bio | 자기소개 (프로필 수정 대상, 최대 300자) | VARCHAR |
| status | 계정 상태 (ACTIVE / DORMANT / WITHDRAWN) | VARCHAR |

> 소셜 계정·리프레시 토큰 테이블은 두지 않는다. 소셜 검증과 세션(access/refresh)은 **Supabase Auth**가 관리하고, 백엔드는 `users.supabase_uid`로 매핑한다.

### diaries (일기)

| 필드 | 설명 | 타입/관계 |
|------|------|----------|
| id | 내부 PK | BIGINT |
| user_id | 작성자 참조 | → users.id |
| content | 일기 본문 | TEXT |
| written_date | 기록 날짜 (user_id + written_date 부분 유니크, deleted_at IS NULL 한정) | DATE |
| deleted_at | 소프트 삭제 시각 (NULL이면 활성) | TIMESTAMP |
