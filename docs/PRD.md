# record MVP PRD

> ⚠️ **이 문서는 고정 명세가 아니라 현재 시점의 기준 스냅샷이다.** 범위·기능·데이터 모델·기술 스택은 개발 진행과 요구사항 변화에 따라 **얼마든지 변경될 수 있다.** 변경 시 `docs/`(api-contract·backend·database 등)와 함께 정합을 맞춘다.

## 핵심 정보

**목적**: 오늘 하루를 글로 남겨 두고 언제든 다시 꺼내볼 수 있는 개인 모바일 기록장. 형식에 얽매이지 않고 그날그날을 가볍게 기록한다.
**사용자**: 매일 짧은 글쓰기로 하루를 남겨 두고 싶은 모바일 사용자

## 사용자 여정

```
1. 앱 실행
   ↓ 로컬 토큰 유무 확인

   [토큰 없음 / 만료] → 로그인 페이지
   [토큰 유효]        → 메인 페이지 (캘린더)

2. 로그인 페이지
   ↓ 소셜 로그인 버튼 탭 (카카오 / 구글)

   [최초 진입 - 미가입] → Supabase 세션 발급 → (백엔드 첫 요청 시 자동 가입) → 메인 페이지
   [재진입 - 기가입]    → Supabase 세션 발급 → 메인 페이지
   [인증 실패]          → 에러 토스트 후 로그인 페이지 유지

3. 메인 페이지 (캘린더)
   ↓ 날짜 탭 / 글쓰기 FAB / 하단 탭 이동

   [날짜 탭 - 확정 기록 있음]        → 글 상세 페이지
   [날짜 탭·FAB - 그 외]            → "글 작성 / 작심삼일 작성" 선택 시트
        ├ 글 작성       → 글 에디터 페이지 (신규 작성 모드)
        └ 작심삼일 작성  → 작심삼일 작성 페이지 (과거 날짜는 비활성)
   [하단 목록 탭]                   → 글 목록 페이지
   [하단 작심삼일 탭]                → 작심삼일 페이지 (진행/성공/실패 + 캘린더)

4. 글 에디터 페이지
   ↓ 사용자 행동

   [등록 버튼]          → DRAFT 저장(미확정·수정가능·미분석) → 메인 페이지
   [오늘을 기억하기 버튼] → 확인 다이얼로그 → 확정(PENDING·감정분석 1회·이후 수정 불가) → 상세 페이지
   [취소 버튼]          → 이전 페이지 (메인 페이지 또는 글 상세 페이지)

5. 글 상세 페이지
   ↓ 사용자 행동

   [수정 버튼] → 글 에디터 페이지 (수정 모드, 기존 내용 로드)
   [삭제 버튼] → 삭제 확인 다이얼로그 → 확인 시 소프트 삭제 → 메인 페이지
   [뒤로가기]  → 이전 페이지 (글 목록 페이지 또는 메인 페이지)

6. 글 목록 페이지
   ↓ 사용자 행동

   [항목 탭]     → 글 상세 페이지
   [캘린더 탭]   → 메인 페이지
   [로그아웃]    → 토큰 삭제 → 로그인 페이지
```

## 기능 명세

### 1. MVP 핵심 기능

| ID | 기능명 | 설명 | MVP 필수 이유 | 관련 페이지 |
|----|--------|------|--------------|------------|
| **F001** | 로그인 (소셜/이메일) | Supabase Auth로 소셜 로그인(카카오/구글) 또는 이메일 가입·로그인(확인 메일 필수), Supabase 세션 발급. 백엔드는 Supabase JWT 검증 + 최초 요청 시 자동 가입(이메일·소셜 동일 경로) (애플은 추후 확장) | 서비스 진입의 유일한 인증 관문 | 로그인 페이지 |
| **F002** | 캘린더 날짜 선택 | 월별 캘린더 표시, 기록 작성 여부를 날짜별로 시각 표시, 날짜 탭으로 작성·조회 진입 | 날짜 기반 기록 작성의 핵심 진입점 | 메인 페이지 |
| **F003** | 기록 작성 | 선택된 날짜에 자유 텍스트 기록 작성(본문 **최대 500자 하드 제한** — LLM 감정분석 비용·품질 가드). 해당 날짜 기록 미존재 시 신규 생성(INSERT). **[등록]**은 DRAFT(미확정·수정가능)로 저장, **[오늘을 기억하기]**는 확정(이후 수정 불가, 감정분석은 확정 시 1회 — F018/F019로 구현됨) | 서비스의 핵심 행위 | 글 에디터 페이지 |
| **F004** | 기록 목록 조회 | 작성된 기록을 날짜 역순으로 나열. 날짜와 내용 미리보기 표시, 커서 기반 무한 스크롤 | 과거 기록 탐색의 기본 수단 | 글 목록 페이지 |
| **F005** | 기록 단건 조회 | 특정 날짜 기록의 전체 내용 표시 | 작성한 글을 읽는 기본 기능 | 글 상세 페이지 |
| **F006** | 기록 수정 | 기존 기록 내용 수정(UPDATE). **DRAFT(미확정) 기록만 수정 가능**, 같은 날짜에 글 에디터 재진입 시에도 DRAFT면 수정으로 처리. **확정('오늘을 기억하기') 후에는 수정 불가**(삭제는 가능 → 같은 날짜 재작성) | 하루 1기록 원칙 구현, 오탈자·내용 변경 지원 | 글 에디터 페이지 |
| **F007** | 기록 삭제 | 기록 소프트 삭제(deleted_at 타임스탬프 기록). 확인 다이얼로그 후 처리. 삭제된 날짜는 다시 작성 가능. 첨부 사진 행·디스크 파일은 즉시 회수(복원 불가) | 잘못 작성된 기록 제거·디스크 공간 절약 | 글 상세 페이지 |
| **F012** | 기록 사진 첨부 (본문 인라인) | 기록 본문에 사진을 **인라인 임베드**(작성 중 `POST /diaries/images`로 즉시 업로드 → 반환 `{url}`을 Delta에 삽입, jpg/png/webp). 바이너리는 스토리지에 저장하고 DB엔 경로만 보관하며 본문 `content`(Delta)가 이미지의 **단일 진실 공급원**. 기록 삭제·수정 시 본문에서 빠진 이미지의 디스크 파일 회수(별도 `diary_images` 1:N 테이블은 폐지) | 글과 함께 그날의 사진 기록 | 글 에디터/상세 페이지 |
| **F011** | 프로필 조회·수정 | 내 프로필(닉네임·프로필 이미지·자기소개 bio) 조회 및 수정. `GET/PUT /users/me`. 가입 시 JIT로 생성된 정보를 사용자가 직접 갱신 | 가입 정보의 사용자 관리·표시 이름 변경 | 프로필 페이지 |

### 2. MVP 필수 지원 기능

| ID | 기능명 | 설명 | MVP 필수 이유 | 관련 페이지 |
|----|--------|------|--------------|------------|
| **F010** | 로그아웃 + 세션 관리 | 로그아웃 시 Supabase signOut(세션 삭제). access 만료 시 Supabase SDK가 자동 갱신, Dio 인터셉터는 Supabase access token 첨부만 담당 | 인증 세션 유지 및 안전한 종료 | 메인 페이지, 글 목록 페이지 |

### 3. 작심삼일(3일 결심) 기능

> 기록(diary)과 **독립된 부가 기능**. 날짜 판정은 전부 **서버 KST(Asia/Seoul) 권위**이며, 완료 체크는 날짜 인자 없이 서버 '오늘'로 판정한다.

| ID | 기능명 | 설명 | 관련 페이지 |
|----|--------|------|------------|
| **F013** | 작심삼일 생성·수정 | **시작일(오늘/미래, 과거 불가) + 할일 제목 + 알림 시각(선택)**으로 3일 결심 생성. 생성 시 3일치 체크(day 1~3, `PENDING`)를 프리생성. 진입은 캘린더 날짜 탭·글쓰기 FAB의 **"글 작성 / 작심삼일 작성" 선택 시트**에서 분기. **진행 중(ONGOING) 결심은 제목·알림 시각만 수정 가능**(`PUT /resolutions/{id}`) — 시작일 변경은 종료일·체크 재계산 복잡도로 미지원(삭제 후 재작성으로 유도) | 작심삼일 작성/수정 페이지 |
| **F014** | 완료 체크 · 성공/실패 판정 | 진행 중 결심의 **오늘 체크를 '완료'(멱등)**. 3일 모두 완료 시 즉시 `SUCCESS`, 하루라도 그 날(KST 자정) 안에 못 누르면 자정 배치가 `MISSED`+결심 `FAILED` 처리 | 작심삼일 상세 페이지 |
| **F015** | 연장 (streak) | **성공한 결심만** 같은 할일로 '다음 3일'을 이어가 새 작심삼일 생성(시작일 `max(이전 종료+1, 오늘)`). `streak_seq`로 연속 N회 표시. 이중 연장 차단 | 작심삼일 상세 페이지 |
| **F016** | 작심삼일 목록·캘린더 조회 | **진행중/성공/실패** 세그먼트 리스트(3일 진행 도트·streak 배지, 커서 페이징) + **월별 캘린더**로 결심을 선택·조회 | 작심삼일 탭 |
| **F017** | 리마인더·완주 푸시 (FCM) | **미완료 시 매일 리마인더**(알림 시각 도래, 하루 1회 멱등), **3일 완주 축하**, 실패 안내를 **FCM 서버 푸시**로 발송. 폰 상단 알림 탭 시 해당 결심 상세로 딥링크 | (시스템 알림 → 작심삼일 상세 페이지) |

### 3.5 감정 분석 · 동적 테마 (LLM)

> 기록 **확정('오늘을 기억하기', DRAFT→PENDING)** 시 멀티모달 LLM이 감정을 분석해 테마·연출을 자동으로 입힌다. 확정 후 **1회만** 분석하며(수정 불가, 재분석은 삭제 후 재작성·재확정), 분석은 비동기(트랜잭션 밖 `@Async`)·실패 시 `NEUTRAL` 폴백. 초기 PRD에서 "MVP 이후 제외"였으나 **구현 완료**되어 승격.

| ID | 기능명 | 설명 | 관련 페이지 |
|----|--------|------|------------|
| **F018** | 감정 분석 (멀티모달 LLM) | 확정 기록의 **순수 텍스트 + 본문 인라인 이미지**를 LLM에 입력해 대표 감정(6종: 기쁨/슬픔/분노/평온/불안/중립)·테마색(배경/글자/강조)·AI 한 줄 코멘트·AI 제목·무드 이모지·감정 점수 분포를 생성. provider 격리(기본 Gemini / Claude·Ollama·Stub, 무키 시 Stub 폴백), 실패 시 NEUTRAL 폴백 | 글 상세 페이지 |
| **F019** | 감정 기반 동적 테마 · 연출 | 분석 결과로 상세·목록에 **감정 배경/글자/강조 색**을 동적 적용하고, **감정 마스코트 영상**·상세 **시네마틱 인트로**·분석 대기 **러닝 로딩 영상**·캘린더 **감정색/이모지**를 렌더링. 분석 중(PENDING)엔 상세에서 3초 폴링 후 자동 전환 | 글 상세·목록·메인 페이지 |

## 메뉴 구조

```
📱 record 내비게이션

━━━━ 비로그인 상태 ━━━━

로그인 페이지 (앱 실행 시 토큰 없으면 자동 리디렉션)
└── 기능: F001 (카카오 / 구글 소셜 로그인)

━━━━ 로그인 후 ━━━━

하단 내비게이션 바 (3탭)
├── 캘린더 탭
│   └── 메인 페이지 - F002 (월별 캘린더). 날짜 탭·글쓰기 FAB → "글 작성 / 작심삼일 작성" 선택 시트로 분기
├── 목록 탭
│   └── 글 목록 페이지 - F004 (날짜 역순 기록 목록, 커서 페이징)
└── 작심삼일 탭
    └── 작심삼일 페이지 - F016 (진행중/성공/실패 세그먼트 리스트 + 월별 캘린더)

상단 앱 바 (로그인 후 공통)
├── 프로필 진입 - F011 (프로필 페이지 → 조회·수정)
└── 로그아웃 버튼 - F010 (토큰 삭제 후 로그인 페이지 이동)

프로필 페이지 - F011 (상단 앱 바 프로필 진입)
├── 조회: 닉네임·이메일·프로필 이미지·자기소개(bio)
└── 수정: 닉네임·프로필 이미지·자기소개 편집 후 저장 (PUT /users/me)

━━━━ 버튼/탭으로 진입하는 화면 ━━━━

글 에디터 페이지 - F003, F006
├── 진입 1: 메인 페이지에서 기록 없는 날짜 탭 → 신규 작성 모드
└── 진입 2: 글 상세 페이지 수정 버튼 탭 → 수정 모드

글 상세 페이지 - F005, F007
├── 진입 1: 메인 페이지에서 기록 있는 날짜 탭
└── 진입 2: 글 목록 페이지 항목 탭

작심삼일 작성 페이지 - F013
└── 진입: 캘린더 날짜 탭·글쓰기 FAB 선택 시트 → "작심삼일 작성" (제목·시작일·알림 시각 입력 후 생성)

작심삼일 상세 페이지 - F014, F015 (+F013 수정 진입)
```

### 모바일 프레임워크

- **Flutter (Dart SDK `^3.10.x`)** - 크로스플랫폼 모바일 앱 (iOS / Android)

### 상태 관리

- **flutter_riverpod ^3.x** - 선언형 상태 관리, AsyncNotifier / Provider 패턴

### 라우팅

- **go_router ^17.x** - 선언형 라우팅, 인증 상태 기반 리디렉션 가드

### 네트워크

- **dio ^5.9.x** - HTTP 클라이언트, 인터셉터로 Supabase access token 첨부(세션 갱신은 Supabase SDK가 담당)

### 모델

- **손 작성 불변 모델(copyWith/fromJson/toJson)** - 초기 설계의 freezed/json_serializable은 **미도입**(코드 생성 보류)하고 손 작성 모델로 대체(ROADMAP Task 004 비고). 재도입 시 확장 가능

### 리치 텍스트 · 미디어

- **flutter_quill ^11.x (+ flutter_quill_extensions)** - 기록 본문 리치 텍스트 에디터(Delta JSON) + 본문 인라인 이미지 임베드. **flutter_localizations** 동반
- **image_picker ^1.x** - 프로필 이미지·본문 인라인 이미지 선택(웹·모바일 공통 바이트 읽기)
- **video_player ^2.x** - 감정 마스코트 영상·러닝 로딩 연출·로그인 화면 영상 재생
- **flutter_shaders ^0.1.x** - 감정 마스코트 투명 배경 합성(불투명 mp4의 색/알파 좌우 패킹 → 프래그먼트 셰이더 `emotion_alpha.frag`로 premultiplied RGBA 합성). 웹은 셰이더 합성 불가로 투명 PNG 포스터 폴백

### 보안 저장

- **flutter_secure_storage ^10.x** - 민감 데이터 보안 저장 (Supabase 세션 토큰은 SDK가 자체 보안 저장소에서 관리)

### 인증 (Supabase Auth)

- **supabase_flutter ^2.x** - Supabase Auth: 소셜 로그인·세션 관리·토큰 자동 갱신
- **google_sign_in ^7.x** - 구글 네이티브 로그인(idToken → Supabase `signInWithIdToken`)
- 카카오는 Supabase `signInWithOAuth`(웹 OAuth, 딥링크 콜백)로 처리 — 별도 카카오 SDK 불필요
- (애플 로그인은 추후 Supabase Apple provider로 확장)

### 알림 (FCM 서버 푸시 — 작심삼일 F017)

- **firebase_core / firebase_messaging ^16.x** - Firebase 초기화 + 푸시 토큰 발급·수신. Firebase 프로젝트 `recorme-c5e1c`, 앱 `com.recorme.app`, `flutterfire configure` 산출물(`firebase_options.dart`·`google-services.json`·gms Gradle 플러그인)
- **flutter_local_notifications ^22.x** - 포그라운드 수신 알림 렌더링(안드로이드 채널 `resolution_reminders`)
- **shared_preferences ^2.x** - 알림 권한 요청 1회 플래그 저장
- 백엔드는 **firebase-admin**(Firebase Admin SDK)으로 발송(`sendEachForMulticast` + 무효 토큰 회수). 서비스계정 키는 `FCM_CREDENTIALS` 환경변수 주입(무키 시 `StubPushService` 폴백)

### 백엔드 및 데이터베이스 (참고)

- **Spring Boot 3.5.x (Java 21)** - REST API 서버, `@Async` 비동기(감정 분석 전용 풀·작심삼일 완주 알림) + `@Scheduled`(감정 분석 PENDING 백스톱 폴러, 작심삼일 자정 실패 배치·리마인더, `FOR UPDATE ... SKIP LOCKED` 다중 인스턴스 안전)
- **MyBatis 3.5.x** - SQL 매퍼 (OFFSET 미사용 커서 페이징)
- **PostgreSQL 18.x** - 관계형 데이터베이스 (uq_diary_user_day 부분 유니크 제약, 감정 분석 V7, 작심삼일 V9/V10)
- **Flyway 11.x** - DB 스키마 버전 관리 (기능별 분할 V1~V10)
- **멀티모달 LLM 감정 분석** - `infra.llm.LlmClient` 추상화로 provider 격리: **Gemini(기본)·Claude(`anthropic-java`)·Ollama(로컬 무키)·Stub(무키 폴백)**. 본문(순수 텍스트)+인라인 이미지를 입력해 감정·테마색·AI 코멘트/제목·무드 이모지를 생성. 실패 시 `NEUTRAL` 폴백
- **firebase-admin** - FCM 서버 푸시 발송(작심삼일 리마인더/완주 알림)
