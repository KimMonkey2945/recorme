# 아이템 생성 프롬프트 가이드 (v2 — 부위별 단독 착용샷 방식)

캐릭터 꾸미기 아이템(모자·안경·상의·하의·신발)을 AI로 생성할 때 쓰는 표준 레시피.

> **v2 방식 확정 (2026-07-15)**: 모든 착용 아이템은 **"캐릭터가 그 아이템만 착용한 완성샷"** 으로 생성한다.
> Claude가 원본과의 픽셀 차이(diff)로 아이템만 추출해 오버레이를 만든다 — AI가 캐릭터 몸·머리에 맞게
> 그렸으므로 위치·핏이 완벽하고, 부위별로 따로 선택·조합할 수 있다.
>
> (v1의 "아이템 단독 제품샷 + 좌표 합성" 방식은 위치가 어색해 **폐기**. 사람 체형 옷 제품샷도 체형
> 불일치로 폐기. 실측 근거: tasks/030 변경 이력 참조.)

---

## 몇 장을 만들어야 하나 — 확장 비용 규칙

두 캐릭터는 **몸 템플릿·프레임(848×1400)이 통일**되어 있다. 그래서:

| 부위 | 필요 수량 | 이유 |
|---|---|---|
| **상의·하의·신발** | **원숭이 1장만** (전 캐릭터 공용) | 몸이 같아서 원숭이에서 추출한 옷이 모든 캐릭터에 맞는다 |
| **모자·안경** | **캐릭터별 1장씩** (원숭이+판다 = 2장) | 머리는 캐릭터 고유 |

→ 새 아이템 세트(모자1·안경1·상의1·하의1·신발1)를 추가할 때 = **7장**.
→ 새 캐릭터를 추가할 때 = 기존 옷 전부 자동 호환, **모자·안경만** 그 캐릭터용으로 재생성.

## 생성 방법

1. 이미지 편집 AI에 **캐릭터 원본**을 첨부한다 — 반드시 최신 정규화본:
   `app/assets/characters/monkey.png` (또는 `red_panda.png`)
2. 원하는 아이템의 **레퍼런스 사진**도 같이 첨부하면 정확도가 올라간다.
3. 프롬프트 (부위별로 [ ] 부분만 교체):

   ```
   첨부한 캐릭터에게 [모자만 / 안경만 / 상의만 / 하의만 / 신발만] 착용시켜줘.
   아이템: [설명 또는 "첨부한 레퍼런스와 같은 ○○"]
   나머지 — 자세, 표정, 얼굴, 몸, 털, 꼬리, 입고 있는 다른 옷, 캔버스 크기, 캐릭터 위치, 배경 —
   는 절대 바꾸지 마. 그 아이템 하나만 추가/교체되어야 해.
   ```

4. 여러 장 뽑아 **"그 아이템 말고는 원본과 똑같은" 컷**을 고른다(얼굴·자세가 변한 컷은 diff에 잡음 → 버릴 것).
5. 배경은 단색(검정 권장)이면 되고 투명일 필요 없다. 워터마크 없는 설정 권장.

### 부위별 주의점

- **모자**: 낮은 프로파일(캡·비니·버킷) 권장 — 위로 높이 솟는 모자는 프레임 상단에서 잘릴 수 있다.
- **안경**: 원본 눈이 살짝 비치는 투명 렌즈도, 불투명 선글라스도 모두 가능.
- **상의**: 원본 흰 티를 완전히 교체한 모습으로. 기장이 엉덩이를 넘으면 하의와 겹칠 때 하의가 많이 가려진다(의도라면 OK).
- **하의**: 원본 검정 반바지를 완전히 교체한 모습으로.
- **신발**: 맨발(발가락)이 완전히 가려지는 디자인.

## 저장 위치와 파일명

```
docs/recormeImo/item/
├── hat/      모자 착용샷      예: black_cap_monkey.png, black_cap_red_panda.png
├── glasses/  안경 착용샷      예: round_glasses_monkey.png, round_glasses_red_panda.png
├── top/      상의 착용샷      예: love_hood_monkey.png        (원숭이만 — 공용)
├── bottom/   하의 착용샷      예: cargo_pants_monkey.png      (원숭이만 — 공용)
├── shoes/    신발 착용샷      예: airmax_monkey.png           (원숭이만 — 공용)
├── room/     방 꾸미기 소품 — 아이템 단독 샷(캐릭터와 무관, v1 방식 그대로)
└── bg/       배경 — 세로 3:4(기준 900×1200) 일러스트, 캐릭터가 서는 중앙~하단은 비워둘 것
```

- 파일명은 **영문 소문자+언더스코어**, 착용샷은 캐릭터 폴더(`wearItem/monkey/`, `wearItem/panda/`)로 구분.
- 추출·합성은 **에셋 제작 시 1회**다. 앱 실행 중에는 완성된 PNG를 겹쳐 그릴 뿐이라 유저 체감 지연이 없다.

## 실행 — 통합 파이프라인 (Claude 없이 직접 가능)

이미지를 폴더에 넣은 뒤 레포 루트에서:

```
powershell -ExecutionPolicy Bypass -File tools\item_pipeline.ps1
```

- 결과 오버레이·썸네일이 `app/assets/items/`에, **검증 이미지가 `docs/recormeImo/checks/`에** 생성된다.
  checks의 이미지를 눈으로 확인하고 이상하면(위치·크기) Claude에게 조정을 요청.
- 방식은 스크립트에 고정: 모자·안경 = 단독샷+착용샷 위치 자동 실측(하이브리드),
  의류 = 착용샷 영역 통째 교체(착용샷 몸이 베이스와 잘 정렬된 캐릭터만 — `clothesRegion` 플래그).
- **새 그룹(새 아이템)**이면 `fake_character_repository.dart`(데모)와 V15 시드(실서비스)에 행 추가 필요 — 이건 코드 작업.
- 앱 반영 확인: `cd app` 후
  `flutter run -d web-server --web-port 8090 --dart-define=USE_FAKE_CHARACTER_REPO=true --dart-define=DEV_BYPASS_AUTH=true`
  → 브라우저 `http://localhost:8090/#/onboarding/character`.

## 자주 실패하는 패턴 (피하기)

| 실패 | 증상 |
|---|---|
| 아이템 단독 제품샷을 좌표로 얹기 (v1) | 모자·안경 위치가 어색, 사람 옷은 체형 불일치 — **폐기된 방식** |
| 착용샷에서 얼굴·자세·다른 옷이 같이 바뀜 | diff에 잡음이 섞여 추출 실패 — 그 아이템만 바뀐 컷을 고를 것 |
| 여러 아이템을 한 장에 같이 착용 | 겹친 부위에 구멍(예: 후드에 가려진 바지 윗단) — 반드시 부위별 1장 |
| 캐릭터 원본이 아닌 다른 이미지 기반 생성 | 몸·위치가 어긋나 정착 불가 |
| 높은 모자(고깔 등) | 프레임 상단에서 잘림 |

## 예외: 풀룩 세트 (차선책)

모자·안경·옷·신발을 **모두 착용한 완성샷 1장**(캐릭터별)을 만들면 "풀룩 세트" 아이템으로 통째 교체할 수
있다(`러브 후드 세트`가 이 방식). 부위별 조합은 안 되지만 제작이 가장 간단하다 — 시즌 한정 코스튬 등에 적합.
