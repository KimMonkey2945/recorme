# Task 032 테스트 시나리오 — 리액션 오버레이 + 월간 회고

> **대상**: F031(기록 리액션), F032(월간 회고·성장).
> **범위 구분**: ✅ 자동 테스트로 이미 검증된 항목은 각 시나리오에 표기했다. 이 문서의 핵심은 **실기기/에뮬 수동 E2E**(자동으로 못 잡는 실렌더·네비게이션·백엔드 연동)다.
> **선행 자동 검증(완료)**: 백엔드 `RetrospectServiceTest`·`RetrospectControllerTest`(Testcontainers) 그린, 앱 `reaction_overlay_test`·`retrospect_test` + 전체 `flutter test` 149개 통과, `flutter analyze` 무경고.

---

## 0. 사전 준비

| 항목 | 명령/조건 |
|---|---|
| 백엔드 기동 | `cd backend && ./gradlew bootRun` (네이티브 PostgreSQL 18 `recorme`) |
| 앱 실행 | `cd app && flutter run` (실기기 Z Flip3 또는 에뮬레이터) |
| 앱↔백엔드 연결 | `adb reverse tcp:8080 tcp:8080` |
| 보상 설정 확인 | `record.character.coin.*`(출석 10·기록 10·연속 7=200 등), `coin-enabled=true` |
| 감정 플래그 | `record.analysis.enabled=false`(기본) — 확정 즉시 `DONE` 보장(리액션 지연 0) |
| 계정 상태 | **신규 계정** 권장(온보딩·빈 회고부터 관통). 기존 계정이면 시나리오 5·6의 수치가 누적값 |

> ⚠️ 리액션·코인은 `character_events` 멱등 게이트를 통과한다. 같은 기록 재확정은 삭제 후 재작성이므로 **새 diaryId**가 되어 새 리액션이 뜨는 게 정상이다(중복 적립 아님 — 별개 이벤트).

## 시나리오 1 — 가입 → 온보딩 → 캐릭터 홈(출석 적립)

**목적**: 로그인 후 첫 화면이 캐릭터 홈이고, 진입 시 출석 코인이 적립되는지.

| # | 절차 | 기대 결과 |
|---|---|---|
| 1-1 | 신규 이메일/소셜로 가입·로그인 | 캐릭터 미선택이므로 **온보딩(`/onboarding/character`)** 으로 이동 |
| 1-2 | 좌우 캐러셀에서 캐릭터(원숭이/레서판다) 확인 → "선택" | 홈(`/`, 탭 index 0)으로 이동, 선택 캐릭터 이름·소개 렌더 |
| 1-3 | 홈 진입 직후 | 하단에 **"출석 완료! 코인 +10"** 스낵바, 상단 코인 칩이 **+10** 반영 |
| 1-4 | 앱을 껐다 같은 날 재진입 | 출석 스낵바 **없음**(하루 1회 — `ATTENDANCE:{date}` 게이트) |
| 1-5 | 하단 탭 확인 | `[캐릭터 홈][캘린더][작심삼일][피드][프로필]` 5개, 홈이 index 0 |

- ✅ 온보딩 렌더·선택·홈 반영은 `character_onboarding_test`·`character_home_test`가 커버. **1-3/1-4(실제 출석 적립)은 백엔드 연동 수동 확인**.

## 시나리오 2 — 기록 확정 → 리액션 오버레이 즉시 등장 (★ F031 핵심)

**목적**: 확정 직후 대기 없이 캐릭터가 반응하고, 코인·대사·ack가 연동되는지.

| # | 절차 | 기대 결과 |
|---|---|---|
| 2-1 | 캘린더에서 오늘 날짜 탭 → 에디터 진입, 본문 작성 | 리치 에디터 정상, 글자수 카운터 동작 |
| 2-2 | 감정 입력 위젯에서 프리셋 1개(예: 기쁨) 선택 | 프리셋 선택 표시(자유 입력과 상호 배타) |
| 2-3 | "오늘을 기억하기" → 확인 다이얼로그 "기억하기" | 상세(`/diary/:id`)로 이동 |
| 2-4 | 상세 진입 순간 | **로딩 스피너·영상 없이 즉시** 리액션 오버레이 등장(캐릭터 + 말풍선) |
| 2-5 | 말풍선 확인 | 맥락 기반 대사 1줄(감정 아님). 캐릭터별 말투 차이(원숭이 느긋 / 레서판다 애쓰는) |
| 2-6 | 코인 획득 카드 | **"코인 +10"** 카드 표시 |
| 2-7 | "확인" 또는 배경 탭 | 오버레이 사라짐, 상세 본문 정상 노출 |
| 2-8 | 홈으로 복귀 | 상단 코인 반영, **미확인 보상 배지 감소**(ack 반영) |

- ✅ 오버레이 렌더·대사·코인 카드·dismiss는 `reaction_overlay_test`가 커버. **2-1~2-3(에디터 타이핑·확정)과 2-4의 실제 백엔드 리액션(`GET /characters/me/reaction`)은 수동 확인**.

## 시나리오 3 — 리액션 엣지 케이스 (빈손 금지·오류 견고·재표시 방지)

| # | 절차 | 기대 결과 |
|---|---|---|
| 3-1 | 코인 적립이 0인 상황(예: 설정에서 `coin.diary=0`으로 임시 조정) 확정 | 코인 카드는 없지만 **대사 1줄은 항상** 표시(빈손 리액션 금지) |
| 3-2 | 백엔드 순간 지연/오류로 리액션 미생성 시 상세 진입 | 오버레이만 생략(또는 기본 대사), **상세 본문 조회는 정상**(오류로 화면이 막히지 않음) |
| 3-3 | 확정 후 뜬 오버레이를 닫고, 캘린더/목록에서 **같은 기록을 다시 열기** | 리액션 오버레이가 **다시 뜨지 않음**(`reaction=1` 없는 일반 진입) |
| 3-4 | 같은 오버레이에서 "확인"을 여러 번 빠르게 탭 | ack 1회 처리, 배지 중복 감소 없음 |
| 3-5 | (백엔드) 같은 diaryId로 리액션 재조회 | 코인 **중복 적립 없음**(멱등 게이트 — 같은 이벤트) |

- ✅ 3-1(대사 항상)·3-3/3-4(재표시·ack)·3-5(멱등)는 `reaction_overlay_test` + `character_journey_test`(멱등 test)가 커버. **3-2(백엔드 오류 시 상세 정상)는 수동 확인 권장**.

## 시나리오 4 — 옷장 코인 구매 → 착용 → 홈 반영

**목적**: 벌어들인 코인으로 아이템을 사고 입으면 홈 캐릭터에 반영되는지.

| # | 절차 | 기대 결과 |
|---|---|---|
| 4-1 | 코인을 아이템 가격 이상 확보(출석·기록 반복). 홈 → "옷장" | 옷장(`/wardrobe`), 5개 슬롯 탭(HAT/GLASSES/OUTFIT/BOTTOM/SHOES) |
| 4-2 | 미보유(잠금) 아이템 탭 | 잠금 시트(`locked_item_sheet`)에 **코인 가격 + 구매 버튼** |
| 4-3 | 잔액 충분 시 구매 | 잔액 차감, 소유 반영, 잠금 해제 |
| 4-4 | 잔액 부족 아이템 구매 시도 | **`COIN_INSUFFICIENT`** 안내(차감 없음, 재시도 가능) |
| 4-5 | 보유 아이템 탭해 착용 → 저장 | 미리보기 반영 → 저장 시 배치 커밋 |
| 4-6 | 홈 복귀 | 캐릭터에 착용 아이템이 **z순 오버레이**로 렌더 |
| 4-7 | 캐릭터 교체(온보딩 재진입 등) | 옷장·착용 유지, variant만 재해석(옷장이 캐릭터를 따라옴) |

- ✅ 옷장 UI·구매·착용은 `wardrobe_test`·백엔드 `CharacterRewardServiceTest`(구매 경합/부족/게이팅)가 커버. **4-6 실렌더 육안 확인**.

## 시나리오 5 — 월간 회고 카드 (★ F032 락인)

**목적**: 이달의 활동이 회고 한 장에 정확히 집계되는지.

| # | 절차 | 기대 결과 |
|---|---|---|
| 5-1 | 홈 → "이달의 기록" | 회고(`/retrospect`), 상단에 "YYYY년 M월" |
| 5-2 | 요약 지표 | **기록 수·최장 연속일·작심삼일 완주·획득 코인** 4개 타일이 실제 활동과 일치 |
| 5-3 | 감정 분포 | 프리셋(색 점 + 한국어 라벨) + **직접 입력 감정(커스텀 라벨)** 이 함께, 많은 순 막대로 렌더 |
| 5-4 | 감정 미입력 기록 존재 시 | 그 기록은 분포에서 제외(합계가 깨지지 않음) |
| 5-5 | 획득 아이템 | 이번 달 구매/해금한 아이템 썸네일·이름 그리드(내 캐릭터 기준 이미지) |
| 5-6 | "이전 달" 탭 | 이전 달로 이동, 다음 달 버튼 **활성화** |
| 5-7 | 이번 달에서 "다음 달" | **비활성(미래 차단)** |
| 5-8 | 기록 0건인 달 | **"이번 달 기록이 아직 없어요"** 빈 상태(깨지지 않음) |
| 5-9 | 로딩/네트워크 오류 | 로딩 뷰 → 오류 시 재시도 뷰 |

- ✅ 요약·감정 분포·빈 달·월 이동/미래 차단은 `retrospect_test` + 백엔드 `RetrospectServiceTest`(집계·빈 달·연속일·IDOR)가 커버. **5-2~5-5의 실데이터 정합은 수동 확인**(백엔드 집계 vs 화면).

## 시나리오 6 — 전 구간 관통 (락인 루프)

**목적**: `가입 → 온보딩 → 작성 → 감정 → 확정 → 리액션 → 코인 → 구매 → 착용 → 홈 → 회고`가 끊김 없이 이어지는지.

| # | 절차 | 기대 결과 |
|---|---|---|
| 6-1 | 신규 계정으로 시나리오 1~2를 순서대로 | 온보딩 → 홈(출석 코인) → 확정 → 리액션(코인) |
| 6-2 | 며칠치 기록을 확정(연속) | 연속 마일스톤(7일=200코인 등) 도달 시 보상함·회고에 반영 |
| 6-3 | 모은 코인으로 시나리오 4(구매·착용) | 홈 캐릭터에 반영 |
| 6-4 | 시나리오 5(회고) | 이달의 기록 수·연속일·감정 분포·획득 코인·획득 아이템이 앞선 활동과 **모두 정합** |
| 6-5 | 작심삼일 1·2일차·완주 수행 | 각 단계 코인 적립(15·15·50), 회고 완주 수 증가 |

- ✅ 자동 E2E `integration_test/character_journey_test.dart`가 (온보딩→홈→회고 + 리액션 즉시 등장 + 코인 멱등 + 구매·착용)을 관통 검증한다. **실기기/에뮬에서 `flutter test integration_test -d <device>` 로 실행**(데스크톱 프로젝트 미구성 환경에서는 미실행 상태).
